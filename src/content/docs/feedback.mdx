This site, the bootstrap instructions, and the agent experience around HDS
improve through your reports. If anything was unclear, broken, or missing
— at any step, on any page —
**[open a GitHub issue on `site-agents`](https://github.com/healthdatasafe/site-agents/issues)**.

Good candidates:

- Your agent misunderstood or ignored part of the instructions.
- A step failed on your machine (include OS, client, and the error).
- You brought a dataset HDS has no item for and had to fall back to notes
  — tell us what the data was, so the data model can grow.
- Wording that confused you (or your agent).

You don't need to be technical: describe what you tried and what happened.
Your agent can draft the issue for you — ask it to.

**This is the single tracker for everything agent-related** — MCP server
bugs, data-model gaps, docs problems alike. File it here on `site-agents`;
we route internally to the right repo and team.

## Don't have a GitHub account?

Filing an issue requires a (free) GitHub account, and your agent can walk
you through creating one in a couple of minutes:

1. Open [github.com/signup](https://github.com/signup).
2. Enter an email address, choose a password and a username.
3. Complete the verification puzzle and enter the launch code GitHub
   emails you.
4. Head to [the new-issue page](https://github.com/healthdatasafe/site-agents/issues/new/choose),
   pick the template that fits, and paste what you and your agent drafted.

No paid plan or profile setup is needed for filing issues.
